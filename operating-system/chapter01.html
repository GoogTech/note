# 操作系统的特征

  • 并发（Concurrence）

  • 共享（Sharing）

    • 互斥共享方式

      • 在一段时间内只允许一个进程访问该资源

    • 同时访问方式

      • 在一段时间内允许多个进程"同时"访问
      • 微观上, 这些进程可能是交替地对该资源进行访问, 即"分时"共享

  • 虚拟（Virtual）

    • 虚拟技术

      • 指把一个物理上的实体（真实存在的）变为若干个逻辑上的对应物（虚拟的）

        • 时分复用技术（如处理器的分时共享）
        • 空分复用技术（如虚拟存储器）

    • 虚拟处理器

      • 指利用多道程序设计把一个物理上的CPU虚拟为多个逻辑上的CPU

    • 虚拟存储器

      • 指用户感觉到（但实际不存在）的存储器

  • 异步（Asynchronism）

    • 指进程的执行不连贯, 并以不可预知的速度向前推进

  • 注意同一时间间隔内（并发）和同一时刻（并行）的区别

# 目标和功能

  • 计算机系统资源的管理者

    • 处理机管理

      • 进程控制
      • 进程通信
      • 死锁处理
      • 处理机调度

    • 存储器管理

      • 内存分配与回收
      • 地址映射
      • 内存保护、共享及扩充

    • 文件管理

      • 文件存储空间管理
      • 目录管理
      • 文件读写管理和保护

    • 设备管理

      • 缓冲管理
      • 设备分配
      • 设备处理
      • 虚拟设备

  • 用户与计算机系统之间的接口

    • 命令接口

      • 利用系统命令来组织和控制作业的执行

        • 联机命令接口（交互式命令接口）

          • 适用于分时或实时操作系统

        • 脱机命令接口（批处理命令接口）

          • 适用于批处理操作系统

    • 程序接口

      • 使用系统调用（广义指令）来请求操作系统为其提供服务

    • GUI

      • 指图形用户界面, 即图像接口
      • 严格来说GUI不是操作系统的一部分

  • 扩充机器

    • 指通常把覆盖了软件的机器称为扩充机器或虚拟机

# 发展

    1. 批处理操作系统

    • 目的

      • 为了解决人机矛盾及CPU和I/O设备之间速度不匹配的矛盾

    • 分类

      • 单道批处理系统

        • 概念

          • 系统对作业的处理是成批进行的, 但内存中始终保持一道作业

        • 特征

          • 自动性
          • 顺序性
          • 单道性

      • 多道批处理系统

        • 概念

          • 允许多个程序同时进入内存并允许它们在CPU中交替运行

        • 特征

          • 多道
          • 宏观上并行
          • 微观上串行

        • 优缺点

          • 优点

            • 资源利用率高
            • 系统吞吐量大

          • 缺点

            • 用户响应时间长
            • 不提供交互能力（常考点）
    1. 分时操作系统

    • 概念

      • 按时间片轮流把处理器分配给各联机作业使用
      • 因此给用户一种独占一台计算机的感觉

    • 特征

      • 同时性
      • 交互性
      • 独立性
      • 及时性
      • 而多道批处理系统是实现作业自动控制而无须人工干预的系统
    1. 实时操作系统

    • 软实时操作系统

      • 指能够接受偶尔违反时间规定且不会引起任何永久性的损害

    • 硬实时操作系统

      • 指某个动作必须绝对地在规定的时刻（或规定的时间范围）发生
    1. 网络操作系统
    1. 分布式操作系统

  • 操作系统的发展历程

  • 区别在于分布OS中的若干个计算机相互协同完成同一任务

# 运行机制

  • CPU的状态划分

    • 用户态（目态）

      • 运行用户自编写的程序

    • 核心态（管态或核心态）

      • 运行操作系统内核程序

    • 区分执行态的主要目的是保护系统程序

      • 用户态 —> 核心态 ：发生在中断产生时
      • 核心态 —> 用户态 ：发生在中断返回用户程序时

  • 操作系统内核包含的内容

    • 时钟管理
    • 中断机制
    • 原语
    • 系统控制的数据结构及处理

# 中断和异常

  • 异常（内中断）

    • 自愿中断——指令中断

    • 强迫中断

      • 硬件故障
      • 软件中断

    • eg

      • 陷入（trap）
      • 程序的非法操作码
      • 地址越界
      • 算数益出
      • 虚存系统的缺页

  • 中断（外中断，也称强迫中断）

    • 外设请求
    • 人的干预

# 系统调用

  • 分类

    • 设备管理
    • 文件管理
    • 进程控制
    • 进程通信
    • 内存管理

  • 执行过程

# 体系结构

  • 大内核

    将操作系统的主要功能模块都作为一个紧密的整体运行在核心态, 从而为应用提供高性能的系统服务.

  • 微内核

    它将内核中最基本的功能(如进程管理等)保留在内核, 而将哪些不需要在核心态执行的功能移到用户态执行,从而降低了内核的设计复杂性.
